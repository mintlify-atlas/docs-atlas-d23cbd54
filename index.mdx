WebGoat is an open-source, deliberately insecure web application created by [OWASP](https://owasp.org/) to help developers, testers, and security professionals learn about web application vulnerabilities in a safe, legal environment. Practice real-world attack techniques, understand how they work, and learn how to defend against them.

<Warning>
  WebGoat is intentionally vulnerable. Never expose it to the internet or run it on a machine connected to sensitive networks. Always disconnect from external networks while using WebGoat.
</Warning>

## What you will learn

WebGoat provides hands-on exercises for every major category of the OWASP Top 10, organized so you progress from understanding a vulnerability concept to actively exploiting it and then learning how to fix it.

## Get started in minutes

<Steps>
  <Step title="Run WebGoat with Docker">
    Pull and start the latest WebGoat container with a single command:
    ```bash
    docker run -it -p 127.0.0.1:8080:8080 -p 127.0.0.1:9090:9090 webgoat/webgoat
    ```
  </Step>
  <Step title="Create an account">
    Open [http://localhost:8080/WebGoat](http://localhost:8080/WebGoat) in your browser and register a new account. Your progress is saved per user.
  </Step>
  <Step title="Choose a lesson">
    Use the left-hand navigation to browse lessons by OWASP category. Start with **Introduction** if you are new to WebGoat.
  </Step>
  <Step title="Complete assignments">
    Each lesson contains reading material followed by interactive assignments. Use the built-in hints if you get stuck.
  </Step>
</Steps>

<Note>
  WebGoat is maintained by OWASP volunteers. If you find bugs or want to contribute new lessons, see the [Contributing Guide](/contributing/guide).
</Note>
